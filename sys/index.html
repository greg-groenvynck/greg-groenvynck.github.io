Case Study

VPN

Algemeen

  • Een Virtueel Particulier Netwerk (Engels: Virtual Private Network, VPN) is een goedkope manier om een Wide Area Network (WAN) te maken maar hierbij behoud je de privé beter in je netwerk.
  • Hier zijn verscheidene voorbeelden (VPN) bv MaTMA: een Machine To Machine netwerk. Hierbij communiceren uw computers bv met elkaar aan de hand van de verbinding tussen een modem aan uw pc en de server. (via een speciale MaTMA simkaart) MaTMA maakt ook gebruik van IPsec (Internet Protocol Security), dat houdt in dat alle IP-pakketten worden beveiligd door encryptie en/of authenticatie (Authenticatie is het proces waarbij iemand nagaat of een gebruiker, een andere computer of applicatie daadwerkelijk is wie hij beweert te zijn.). MaTMa maakt ook gebruik van GRE-tunnel. Hierbij wilt men zeggen dat ze gebruik maken van het inwerken van netwerkpakketten in een ander netwerkpakket. GRE staat ook wel bekend als Generic Routing Encapsulation.
  • De verzonden data kunnen het best beveiligd worden zodat de integriteit, autorisatie en authenticiteit van de data privé blijft. Hierbij wil ik zeggen dat niet zomaar iedereen kan zien van waar dit komt (dankzij GRE) en wat er in staat.
  • VPN-gebruiker kan onbeperkt IP-adressen voorzien, vaak is er maar bij pc naar pc 1 fysieke verbinding maar bij VPN kunnen verschillende verbindingen worden opgesteld.
  • Door encryptie en controlemaatregelen (bvb CRC : Cycli Reduncy Check = fout dedecteren via verificatie van data en via controle van code) kan een goed uitgewerkt VPN toch de integriteit, autorisatie en authenticiteit van de verzonden data verzorgen. Verder moet er ook rekening gehouden worden met wetten die van kracht zijn omtrent de privacy van data. bv de wet van Bescherming van de persoonlijke levenssfeer in de sector elektronische communicatie, zegt dat: "De lidstaten moeten door hun nationale wetgeving het vertrouwelijke karakter van de communicatie via openbare elektronische communicatienetwerken moeten garanderen.", juli 2002 wat heeft voor gezorgd dat er een verplichting is om een VPN beveiligd op te stellen.

Voordelen

  • Door de brede beschikbaarheid van het internet zal het private netwerk algemeen toegankelijker worden.
  • Je kan denken dat deze makkelijk te "hacken" zijn maar dat is ook niet 100% waar aangezien er makkelijk plaats voorzien is om een firewall aan te sluiten op het systeem waarop deze draait.
  • Zeker voor opkomende bedrijven is dit een zeer goedkope en makkelijke manier van computers met elkaar verbinden zonder het fysieke aspect ervan te moeten betalen (geen cash).
  • En dan hebben we ook nog dat je een bepaalde pc kan overnemen (dat is niet zo in alle gevallen wegens veiligheid). Dat is op zich zeer handig als bepaald personeel niet op werk geraakt en bv van thuis uit kan werken.

Nadelen

  • Er zijn niet echt veel nadelen. Buiten dat het niet voor de hand liggend is.
  • En niet zomaar voor iedereen een voordelend systeem is.
  • Het kan wel zijn dat als er een fout ligt bij een provider van de cliënt die op het systeem wilt, dat men niet via VPN op het gewilde systeem geraakt. Vaak ligt de fout aan de cliënt zelf.

Conclusie

  • VPN is een zeer goede manier van werken van thuis uit of van op verplaatsing.
  • Voor bedrijven is het zeker een goede manier van kosten besparen en productiviteit te verhogen (geen fysiek lijnen leggen en bv van thuis uit werken).
  • Zelf zou het misschien interessant zijn om op school in te voeren om dan van thuis uit aan bepaalde documenten te geraken maar ik kan zien dat dit misschien voor veiligheidsmaatregelen moeilijker is om te doen.

IT-Tapa

Google forms

Wat precies?

  • Onze bedoeling was om leerkrachten een alternatieve manier te laten zien van hoe ze testen kunnen geven aan de leerlingen.
  • Een efficiëntere manier om te verbeteren en makkelijk bij te houden aangezien het digitaal staat opgeslagen.

Waarom dit onderwerp?

  • We vonden dat dit misschien wel interessant was voor bepaalde leerkrachten met theorie vakken. Het verbeteren zou hierdoor vlotter kunnen gaan (standaards verbetersleutels bijvoorbeeld) en hun tijd besparen.
  • Aangezien dat alle leerkrachten al standaard een Immalle adres hebben, is het gemakkelijk voor leerkrachten om hiermee te werken. Ze kennen reeds de werking en er moet geen nieuwe site of tool gecreëerd worden.

Resultaat

  • 6 leerkrachten + directeur zijn komen volgen.
  • Nu weten we ook een beetje hoe het voelt om leerkracht te zijn.

Versies

  • versie VirtualBox: Versie 5.0.14 r105127
  • versie Xubuntu: Xubuntu 14.0.4
  • Versie Windows server: 2012 R2
  • versie host-machine: Windows 7/10

PHP: (Mario)

    Bij het creëren van de testopstelling hebben we vooral met Linux Xubuntu gewerkt.

    Een mogelijk alternatief was de Windows server met IIS, maar hier doken enkele problemen op in verband met het maken van de connectie tussen de database en de webpagina.

    Om de webpagina lokaal te kunnen testen op de Xubuntu hebben we gebruik gemaakt van apache2.

    Handig hieraan is dat het samen met de Virtuele machine word opgestart en dat je bestanden altijd gehost worden.

    De bestanden die gehost worden plaats je in de var/www/html map van het Device “Computer”.

    Om in deze map te geraken moet je wel een programma installeren via Sudo apt-get install Nautilus.

    Eenmaal je bestander hier gezet kan je ze ook nog steeds aanpassen.

MYSQL (Wout)

    Als testopstelling hebben we ook het opensource-managementsysteem voor databases namelijk MYSQL gebruikt.

    Omdat we willen testen dat wanneer je een klasnummer ingeeft de gebruikt een leerling terugkrijgt , moeten we met een database werken.

    We hebben in MYSQL Workbench een database ‘testopstelling’ met een tabel ‘klasgenoten’ gemaakt met hierin verschillende query’s.

    Het werken met MYSQL is in het algemeen goed verlopen. We hebben kleine problemen gehad met het installeren van PhpMyAdmin was moeilijk verlopen.

    Nadat dit was gelukt hebben we geen enkel probleem meer gehad.

    De code in MYSQL zelf was ook niet moeilijk omdat we dit met het vak software ook al veel met SQL bezig geweest zijn.

    Je kan zelfs een code genereren zodat je zelf de code niet moet schrijven.

    Dit heeft wel een nadeel omdat je wanneer je een fout hebt of een query moet toevoegen, je geen idee hebt hoe dit werkt en dus ook een nieuwe tabel zal moeten maken.

Linken van PHP met MYSQL (Greg)

    Php en mysql kan werken met een mysql database door het gebruik van MySQL extension of een PDO (PHP data objects).

    Zowel MySQL en PDO hebben hun voordelen. MySQLi zal alleen werken met SQL databases en PDO kan met 12 verschillende databases werken.

    Als je moet switchen tussen verschillende databases is het het best om dus PDO te gebruiken aangezien de verschillende databases die je kan gebruiken.

    Met MySQLi moet je de hele code herschrijven wat een voordeel is en ook een nadeel.

    Het voordeel is dat je zelf de specificaties kan schrijven het nadeel is dit vraag veel kennis werk en moeite.

    Beiden zijn object georiënteerd maar MySQLi bied een API aan.
